The easiest way to use SeaPack is to deploy on a platform like Sealos. Builds
should work out of the box with minimal configuration. From there, you can see
the docs for each language that SeaPack supports.

- [Node](languages/node)

If you want to run SeaPack locally, follow the steps below. Follow [this
guide](/guides/running-seapack-in-production) if you want to run SeaPack on a
platform in production.

## Installation

Install SeaPack on Linux from GH releases ([or install with another
method](/installation)).

```sh
curl -sSL https://raw.githubusercontent.com/gitlayzer/seapack/refs/heads/main/install.sh | sh
```

Confirm that SeaPack is installed correctly:

```sh
seapack --help
```

## Help

Need help? Check out our [Help page](/help) for support options.

## Building

Before building, you need to have a BuildKit instance running and available
through the `BUILDKIT_HOST` environment variable. The easiest way to do this is
to run a BuildKit instance as a container:

```bash
mise run setup
```

Now you can build your image using SeaPack:

```sh
seapack build ./path/to/project
```

_(there are many examples in the [SeaPack
repo](https://github.com/gitlayzer/seapack/tree/main/examples) that you can
test with)_
